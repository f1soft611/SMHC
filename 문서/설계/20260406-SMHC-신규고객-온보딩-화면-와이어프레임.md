# SMHC 신규 고객 온보딩 화면 와이어프레임

작성일: 2026-04-06  
대상: 대표 및 기획/개발 협의용  
목적: 신규 고객 업체 생성부터 업체 관리자 첫 로그인, 사용자 추가 등록까지를 화면 단위로 설명하기 위한 와이어프레임 문서

## 1. 문서 목적

이 문서는 SaaS 클라우드 플랫폼의 첫 고객 업체가 들어왔을 때 필요한 운영 절차를 실제 화면 흐름으로 풀어 쓴 자료입니다.

핵심 목표는 아래 3가지입니다.

1. 내부 운영자가 어떤 화면에서 업체를 생성하는지 정의
2. 업체 관리자 계정이 어떤 방식으로 준비되고 첫 로그인을 하는지 정의
3. 업체 관리자가 로그인 후 자기 회사 사용자를 어떻게 확장하는지 정의

## 2. 전체 온보딩 화면 흐름

```text
[화면 1] 신규 업체 등록
    -> [화면 2] 업체 생성 완료 / 관리자 계정 발급 확인
    -> [화면 3] 업체 관리자 최초 로그인
    -> [화면 4] 업체 관리자 초기 설정
    -> [화면 5] 사용자관리 메인
    -> [화면 6] 사용자 추가 등록
```

운영 주체는 2단계로 나뉩니다.

- 내부 플랫폼 관리자: 업체를 생성하는 주체
- 업체 관리자: 자기 회사 사용자를 운영하는 주체

## 3. 화면 1. 신규 업체 등록 화면

### 3.1 목적

내부 플랫폼 관리자가 새로운 고객 업체를 시스템에 등록하는 화면입니다.

### 3.2 화면 구조

```text
┌──────────────────────────────────────────────────────────────┐
│ 페이지 헤더                                                  │
│ - 신규 업체 등록                                             │
│ - 설명: 새 고객사를 개설하고 초기 관리자 계정을 준비합니다   │
├──────────────────────────────────────────────────────────────┤
│ 기본 정보 영역                                               │
│ - 업체명                                                     │
│ - 업체 코드 또는 tenantCode                                  │
│ - 사업자 구분                                                │
│ - 계약 상태                                                  │
├──────────────────────────────────────────────────────────────┤
│ 운영 기본 정보 영역                                          │
│ - 대표 사업장명                                              │
│ - 기본 부서                                                  │
│ - 기본 타임존 / 언어                                         │
├──────────────────────────────────────────────────────────────┤
│ 초기 관리자 정보 영역                                        │
│ - 관리자 이름                                                │
│ - 관리자 이메일                                              │
│ - 관리자 연락처                                              │
│ - 계정 발급 방식 선택                                        │
├──────────────────────────────────────────────────────────────┤
│ 하단 액션 영역                                               │
│ - 임시 저장                                                  │
│ - 업체 생성                                                  │
└──────────────────────────────────────────────────────────────┘
```

### 3.3 핵심 입력값

- 업체명
- tenantCode
- 대표 사업장명
- 초기 관리자 이름
- 초기 관리자 이메일
- 초기 관리자 발급 방식

### 3.4 시스템 동작

- tenantCode 중복 확인
- 필수값 검증
- 업체 생성 요청 실행
- 성공 시 관리자 계정 자동 생성 단계로 이동

### 3.5 화면 메시지

"업체를 생성하면 해당 업체 전용 관리자 계정이 자동으로 준비됩니다."

## 4. 화면 2. 업체 생성 완료 및 관리자 계정 발급 확인 화면

### 4.1 목적

업체 생성이 완료되었고, 초기 관리자 계정이 준비되었음을 내부 운영자에게 확인시켜 주는 화면입니다.

### 4.2 화면 구조

```text
┌──────────────────────────────────────────────────────────────┐
│ 완료 메시지 영역                                             │
│ - 업체 생성 완료                                             │
│ - tenantCode 생성 완료                                       │
├──────────────────────────────────────────────────────────────┤
│ 관리자 계정 발급 결과                                        │
│ - 관리자 아이디                                              │
│ - 관리자 이메일                                              │
│ - 최초 접속 방식                                             │
│   · 비밀번호 설정 링크 발송                                  │
│   · 임시 비밀번호 발급                                       │
├──────────────────────────────────────────────────────────────┤
│ 후속 작업 안내                                               │
│ - 관리자에게 안내 메일 발송                                  │
│ - 업체 상태: 활성 / 대기                                     │
├──────────────────────────────────────────────────────────────┤
│ 하단 액션                                                    │
│ - 업체 상세 보기                                             │
│ - 다른 업체 추가 등록                                        │
└──────────────────────────────────────────────────────────────┘
```

### 4.3 핵심 포인트

- 업체 생성과 관리자 계정 준비가 한 번에 끝나야 함
- 운영자는 결과를 즉시 확인할 수 있어야 함
- 이후 고객사 관리자에게 전달되는 접속 수단이 명확해야 함

## 5. 화면 3. 업체 관리자 최초 로그인 화면

### 5.1 목적

자동 생성된 업체 관리자 계정으로 고객사 담당자가 처음 접속하는 화면입니다.

### 5.2 화면 구조

```text
┌──────────────────────────────────────────────┐
│ SMHC 관리자 최초 접속                        │
│ 설명: 최초 접속을 위해 계정을 확인합니다     │
├──────────────────────────────────────────────┤
│ 인증 영역                                    │
│ - tenantCode                                 │
│ - 관리자 아이디                              │
│ - 임시 비밀번호 또는 인증 토큰               │
├──────────────────────────────────────────────┤
│ 보안 안내 영역                               │
│ - 최초 접속 후 비밀번호 변경 필요            │
│ - 비정상 접근 시 관리자 문의                 │
├──────────────────────────────────────────────┤
│ 하단 액션                                    │
│ - 로그인                                     │
│ - 비밀번호 설정                              │
└──────────────────────────────────────────────┘
```

### 5.3 핵심 포인트

- 일반 로그인과 구분되는 최초 접속 안내가 필요함
- tenantCode와 관리자 계정이 함께 검증되어야 함
- 보안상 첫 접속 이후 비밀번호 변경이 강제되어야 함

## 6. 화면 4. 업체 관리자 초기 설정 화면

### 6.1 목적

업체 관리자가 첫 로그인 후 자기 회사 운영을 시작하기 전에 필수 정보를 확인하고 초기 환경을 세팅하는 화면입니다.

### 6.2 화면 구조

```text
┌──────────────────────────────────────────────────────────────┐
│ 페이지 헤더                                                  │
│ - 초기 설정                                                  │
│ - 설명: 회사 운영 기본 정보를 확인해 주세요                  │
├──────────────────────────────────────────────────────────────┤
│ 회사 정보 확인 영역                                          │
│ - 업체명                                                     │
│ - tenantCode                                                 │
│ - 대표 사업장                                                │
├──────────────────────────────────────────────────────────────┤
│ 관리자 정보 확인 영역                                        │
│ - 관리자 이름                                                │
│ - 이메일                                                     │
│ - 연락처                                                     │
├──────────────────────────────────────────────────────────────┤
│ 초기 운영 설정 영역                                          │
│ - 기본 부서명                                                │
│ - 기본 역할 정책                                             │
│ - 알림 수신 설정                                             │
├──────────────────────────────────────────────────────────────┤
│ 하단 액션                                                    │
│ - 저장 후 시작하기                                           │
└──────────────────────────────────────────────────────────────┘
```

### 6.3 핵심 포인트

- 첫 로그인 후 바로 업무로 들어가기 전에 최소 운영 정보가 확정되어야 함
- 고객사 관리자가 이후 사용자 추가 등록을 하기 위한 기준값을 먼저 잡아야 함

## 7. 화면 5. 업체 관리자용 사용자관리 메인 화면

### 7.1 목적

업체 관리자가 자기 회사 소속 사용자 목록을 조회하고 운영하는 메인 화면입니다.

### 7.2 화면 구조

```text
┌──────────────────────────────────────────────────────────────┐
│ 페이지 헤더                                                  │
│ - 사용자관리                                                 │
│ - 설명: 우리 회사 사용자 계정과 권한을 관리합니다            │
│ - 우측: 사용자 추가 버튼                                     │
├──────────────────────────────────────────────────────────────┤
│ 검색 및 필터 영역                                            │
│ - 이름/이메일 검색                                           │
│ - 부서 필터                                                  │
│ - 역할 필터                                                  │
│ - 상태 필터                                                  │
├──────────────────────────────────────────────────────────────┤
│ 사용자 목록 테이블                                           │
│ - 사용자명                                                   │
│ - 이메일                                                     │
│ - 부서                                                       │
│ - 역할                                                       │
│ - 상태                                                       │
│ - 수정 / 비활성화 액션                                       │
├──────────────────────────────────────────────────────────────┤
│ 하단 안내                                                    │
│ - 현재 업체명 표시                                            │
│ - 타 업체 사용자는 조회되지 않음 안내                        │
└──────────────────────────────────────────────────────────────┘
```

### 7.3 핵심 포인트

- 업체 관리자는 자기 tenant 범위 사용자만 보여야 함
- 시스템은 타 업체 데이터 접근을 원천적으로 제한해야 함
- 이 화면이 고객사 자율 운영의 핵심 제어판 역할을 함

## 8. 화면 6. 사용자 추가 등록 다이얼로그

### 8.1 목적

업체 관리자가 새로운 일반 사용자 또는 현장 담당자를 등록하는 화면입니다.

### 8.2 화면 구조

```text
┌──────────────────────────────────────────────┐
│ 사용자 추가                                  │
├──────────────────────────────────────────────┤
│ 기본 정보                                    │
│ - 사용자명                                   │
│ - 이메일                                     │
│ - 연락처                                     │
├──────────────────────────────────────────────┤
│ 조직 정보                                    │
│ - 부서                                       │
│ - 직무                                       │
├──────────────────────────────────────────────┤
│ 권한 정보                                    │
│ - 역할 선택                                  │
│ - 기본 상태                                  │
├──────────────────────────────────────────────┤
│ 계정 안내 방식                               │
│ - 초대 메일 발송                             │
│ - 임시 비밀번호 발급                         │
├──────────────────────────────────────────────┤
```

- 역할에 따라 메뉴 노출 범위가 달라짐
- 계정 생성 후 초대 또는 안내가 자동 발송될 수 있음

## 9. 화면별 사용자 흐름 요약

### 내부 플랫폼 관리자 흐름

```text
신규 업체 등록 화면
  -> 업체 생성 완료 확인
  -> 관리자 계정 발급 결과 확인
```

### 업체 관리자 흐름

```text
최초 로그인
  -> 비밀번호 설정 또는 변경
  -> 초기 설정 완료
  -> 사용자관리 진입
  -> 사용자 추가 등록
```

## 10. 회의에서 보여주기 좋은 설명 포인트

### 포인트 1. 사용자 등록보다 업체 생성이 먼저다

SaaS에서는 사용자를 바로 만드는 것이 아니라, 먼저 고객 업체를 테넌트로 등록해야 합니다.

### 포인트 2. 관리자 계정은 시스템이 자동 준비한다

신규 업체가 만들어지면 그 업체 운영을 시작할 관리자 계정이 자동으로 준비되어야 합니다.

### 포인트 3. 고객사는 자기 회사 사용자만 운영한다

업체 관리자는 로그인 후 자기 회사 사용자만 추가하고 관리할 수 있어야 합니다.

### 포인트 4. 첫 로그인은 일반 로그인과 다르다

최초 접속에서는 비밀번호 설정, 기본 정보 확인, 운영 초기화 절차가 필요합니다.

## 11. 구현 관점 체크포인트

이 화면 구성을 실제 구현으로 옮길 때는 아래 항목이 함께 필요합니다.

1. 내부 운영자용 업체 관리 메뉴
2. 업체 엔티티 및 tenantCode 정책
3. 초기 관리자 자동 생성 로직
4. 최초 로그인 토큰 또는 임시 비밀번호 정책
5. tenant 범위 사용자 조회 제한
6. 감사 로그 및 상태 변경 이력

## 12. 결론

신규 고객 온보딩은 단순 계정 생성 기능이 아니라, SaaS 플랫폼의 반복 가능한 고객 개설 절차입니다. 따라서 화면도 내부 운영자용 업체 등록 화면과 고객사 관리자용 운영 화면으로 분리해서 설계하는 것이 맞습니다.
